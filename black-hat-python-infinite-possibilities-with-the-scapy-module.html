Black Hat Python: Infinite possibilities with the Scapy Module

Wed 22 October 2014 Category Networking Tags Python scapy Book traceroute PCAP ARP_poisining sniffing Wireshark packet_header

This is a comprehensive review about one of my favorites libraries in Python: Scapy, a very powerful interactive packet manipulation program.

Scapy is able to forge and decode packets of several protocols, send and capture them, match requests and replies, and much more. It can be used to handle most network tasks such as scanning, tracerouting, probing, attacks, or network discovery, to name a few.

The topics covered in this post are the following:

  • A Quick Intro to Scapy,
  • Sniffing with Scapy,
  • Fuzzing with Scapy,
  • Plotting with Scapy,
  • Stealing plain text email data,
  • ARP poisoning a machine, and
  • Processing PCAP files.

All right, install Scapy in your machine,

$ pip install scapy

test if it's working (Scapy can be used iteratively),

$ scapy
Welcome to Scapy (2.2.0)
>>> ls()    ---> list protocols
>>> lsc()   ---> list commands
>>> conf    ---> Display configurations
>>> help(sniff)     --> Help for a specific command

and let the fun begin!

Scapy 101

A Simple Packet and its Headers

Let's start specifying the source and then the destination of an IP header, which is a layer 3 protocol. These are the two ways one can do this:

>>> ip = IP(src="192.168.1.114")
>>> ip.dst="192.168.1.25"
>>> pritnt ip
<IP  src=192.168.1.114 dst=192.168.1.25 |>

Scapy builds packets by the layers and then by the fields in each layer. Each layer is nested inside the parent layer, represented by the < and > brackets.

Now let's add a layer 4 protocol, such as TCP or UDP. To attach this reader to the previous, we use the the operator /:

>>> ip/TCP()
<IP  frag=0 proto=tcp src=192.168.0.1 dst=192.168.0.2 |<TCP  |>>
>>> tcp=TCP(sport=1025, dport=80)
>>> (tcp/ip).show()

We could even go further, adding layer 2 protocols such as Ethernet or IEEE 802.11:

>>> Ether()/Dot1Q()/IP()

>>> Dot11()/IP()

Sending a Packet

Scapy's function send tells Scapy to send a single packet to some IP destination (it's a layer 3 send, so it decides the routing based on local table). In another hand, the method sendp is a layer 2 send. The operator / is used as a composite operator between two layers. This operator has been used as a composition operator between layers. The lower layer can have more values overloading according to the upper later.

The functions with a ‘1’ in them mean that Scapy will send the specified packet and end after receiving 1 answer/response instead of continuing to listen for answers/responses.

For example, let us create an ICMP packet with some message:

from scapy.all import *
packet = IP(dst="192.168.1.114")/ICMP()/"Helloooo!"
send(packet)
packet.show()

Notice that the method show() displays details about the packet:

scapy also has the ability to listen for responses to packets it sends, such as ICMP echo requests (pings). We can build an IP packet carrying an ICMP header, which has a default type of echo request, and use the sr() (send/receive) function to transmit the packet and record any response.

sr(IP(dst="packetlife.net")/ICMP()) Begin emission: Finished to send 1 packets. * Received 1 packets, got 1 answers, remaining 0 packets (, )

What if we want to send and listen for responses to multiple copies of the same packet? We can use the srloop() function and specify a count of packets to send.

srloop(IP(dst="packetlife.net")/ICMP(), count=3)

Receiving a Packet

There are two types of packet receiving, based on the network layer: Layer 3: sr returns the answered and unanswered packets, while sr1 only returns answered and sent packets. Layer 2: srp returns the answered and unanswered packets, while srp1 only returns answered and sent packets.

from scapy.all import *
output=sr(IP(dst='google.com')/ICMP())
print '\nOutput is:' + output
result, unanswered=output
print '\nResult is:' + result

We can create custom packets to perform port scanning. For example, we can create a TCP/IP packet with the TCP flag set to 'S' (SYM) for ports 1-1024:

res, unans = sr( IP(dst='192.168.1.114')/TCP(flags='S', dport=(1, 1024)))

It will take a couple of minutes to scan. We can check the output with:

res.summary()

TCP Three-way Handshake

Scapy allows you to craft SYN request and match the corresponding returned SYN/ACK segment.

First we create an instance of an IP header. Then we define an instance of the TCP header called SYN. We send this and capture the server's response so that we can extract the server's TCP sequence number.

The code below will send one packet and match the first response with the command sr1 (layer 3). The notation SYNACK.seq extract the TCP sequence number from the server and increment it by 1.

Then we create a new instance of the TCP header ACK, which now has the flag A. We place the acknowledement value for the server thre. We then send everything (this command does not listen for a response).

ip = IP(src='192.168.1.114', dst='192.168.1.25')
SYN = TCP(sport=1024, dport=80, flags='S', seq=12345)
packet = ip/SYN
SYNACK = sr1(packet)
ack = SYNACK.seq + 1
ACK = TCP(sport=1024, dport=80, flags='A', seq=12346, ack=ack)
send(ip/ACK)

All we have to do now is create the segment with no TCP flags and payload and send it:

PUSH = TCP(sport=1024, dport=80, flags='', seq=12346, ack=ack)
data = "HELLLO"
send(ip/PUSH/data)

However, there is a complications. There is an impediment to crafting TCP sessions with Scapy because it tries to circumvent the native TCP/IP stack. The host is unaware that Scapy is sending packees, and the native host receives an unsolicited SYN/ACK that is not associated with any open session/socket it knows. Therefore, the host resets the connection when it receives the SYN/ACK.

One solution for this is to use the host's firewall with iptables to block the outbound resets:

$ iptables-A OUTPUT-p tcp -d 192.168.1.104 -s 192.168.1.103 --dport 80 --tcp-flags RST RST -j DROP

This drops all outbound packets that are TCP and destined for IP 192.168.1.104 from source IP 192.168.1.103 to destination port 80, where the flags field reset bit should be examined. If it is set, it drops the packet. This doesn't prevent the originating host from generating a reset each time it receives a packet from this session, but it blocks it from leaving the host, silecing the resets.

The Routing Table

To look to the routing table of our machine we can just print the Scapy's command conf.route:

127.0.0.0       255.0.0.0       0.0.0.0         lo              127.0.0.1

192.168.1.0     255.255.255.0   0.0.0.0         wlp1s0          192.168.1.114

Scapy allows us to include a specified route to this table, so any packet intended to some specified host would go through the specified gateway:

conf.route.add(host='192.168.118.2', gw='192.168.1.114')

Finally, to return to the original configuration, we use conf.route.resync().

Other Cool Stuff

Network Functions

Scapy can perform simple networking functions such as traceroute:

print scapy.traceroute('www.google.com')

And commands for network based attack such as arpcachepoison and srpflood.

To discover hosts on the local Ethernet, we can use arping

print arping('192.168.1.114')

The Command Method

If you’re wanting to see exactly how to re-create a packet that’s been sniffed or received, Scapy has a packet method for you! Using the .command() packet method will return a string of the command necessary to recreate that packet, like this:

The Sniff Function

Packet sniffing can be done with the function [sniff]. For example:

p = sniff(filter='icmp', iface='eth1', timeout=10, count=5)
print p.summary()

Now, to see the output in real time, we use the lambda function for the summary:

p = sniff(filter='icmp', iface='eth1', timeout=10, count=5,  prn=lambda x:x.summary())

>>> a=sniff(filter="tcp and ( port 25 or port 110 )",

We can sniff and do passive OS fingerprinting:

load_module("p0f")
>>> p0f(p)

Fuzzing

The function fuzz() is able to change default values by random values, so we can build fuzzing templates and send them in loop.

In the following example, the IP layer is normal, and the UDP and NTP layers are fuzzed. The UDP checksum will be correct, the UDP destination port will be overloaded by NTP to be 123 and the NTP version will be forced to be 4. All the other ports will be randomized:

>>> send(IP(dst="target")/fuzz(UDP()/NTP(version=4)),loop=1)
................^C
Sent 16 packets.

Here is a DNS fuzzer:

send(IP(dst='192.168.1.114')/UDP()/fuzz(DNS()), inter=1,loop=1)
